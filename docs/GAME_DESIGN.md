# GAME_DESIGN — おじさんクエスト（ojisan-quest）骨子

> ドラゴンクエスト風の"構造"をリスペクトしたパロディRPG。中身は全ておじさんオリジナル。
> これは"種"。新チャットで本人と詰めて確定させる（`_starter-kit/README.md` フェーズ0）。

## パロディの線引き（厳守）
- 借りる：王道RPGの**構造**（フィールド→町→ダンジョン→ボス、コマンド戦闘、レベル制、宿屋/道具屋、復活の呪文的なセーブ）と、愛あるオマージュのトーン。
- 借りない：ドラクエ固有のモンスター（スライム等）・呪文名（メラ等）・地名・ロゴ・音楽・固有の見た目。
- 敵・技・呪文・地名・UI文言は**全ておじさんオリジナル**。

## ジャンル / コアループ
- ターン制コマンドバトルRPG。
- ループ：フィールド移動 → エンカウント → コマンド戦闘（たたかう／とくぎ／どうぐ／にげる 等の"おじさん版"）→ 経験値/お金/アイテム → 町で補給 → ダンジョン → ボス。

## 世界観（流用前提・コミカル哀愁）
- 主人公：おじさん（A案）。「会社という名の魔境」を巡る、出世と残業の哀愁をネタにした冒険、の骨子。
- 敵：既存ロスター流用（イモムシ社員・果物メガネ女子・バナナ筋肉女子・大仏豚ボス 等）。
- "王様"ポジション＝社長、"教会"＝喫茶店、みたいなおじさん的読み替えで王道構造をパロる。

## 流用 vs 新規
| 区分 | 流用（発注不要） | 新規発注 |
|---|---|---|
| キャラ | 主人公 idle/attack/hurt、敵ロスター、アイテム | NPC（町の人/店主/社長）、追加敵、中/ラスボス |
| 画面 | — | バトル背景3〜4種、フィールドのタイルセット、UI（コマンド枠/HPバー/アイコン） |

## データ駆動の単位（`scaffold/content.example.js` の RPG 例に沿う）
- `ENEMIES`（hp/atk/def/exp/gold/skills/sprite）, `SKILLS`（kind/power/target/element）,
  `ENCOUNTERS`（エリア→敵編成/出現率）, `DIALOGUE`/`SCENARIO`, `ITEMS`。エンジンは薄く、内容はデータで増やす。

## ★新チャットで決めること（open questions）
1. **物語の前提**（誰が・なぜ・どこを巡るか。1段落で言い切る）。
2. **戦闘の"一捻り"**（ただの殴り合いにしない仕掛け。例：弱点=おじさんの本音／満員電車ゲージ／名刺コンボ 等）。
3. **マップ構造**（ワールドマップ＋町＋ダンジョン or 一本道。最初は小さく）。
4. **成長**（レベル制／装備／とくぎ習得）。
5. **セーブ**（チェックポイント or どこでも）。
6. 技術（Phaser+Vite で確定か。UIが多いので検討）。
