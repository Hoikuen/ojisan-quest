# GAME_DESIGN — おじさんクエスト（ojisan-quest）骨子

> オリジナルの**シンプルなターン制コマンドRPG**。特定作品のパロディではない。作りやすさ最優先。
> これは"種"。新チャットで本人と詰めて確定させる（`_starter-kit/README.md` フェーズ0）。
> 共通ルール：有名作の固有キャラ/名称/ロゴ/音楽/見た目は使わない（全部おじさんオリジナル）。

## ジャンル / コアループ
- ターン制コマンドバトルRPG（正面ビュー・行動順は素早さ順・ATBなし＝実装が単純で堅牢）。
- ループ：フィールド移動 → エンカウント → コマンド戦闘（たたかう／とくぎ／どうぐ／にげる）→ 経験値/お金/アイテム → 町で補給 → ダンジョン → ボス。

## 作りやすさのための最小構成（まずここを動かす）
- **主人公おじさん1人パーティ**で開始（仲間は後で足せるデータ構造に）。
- 敵は前方に静止スプライト表示。被弾でフラッシュ、撃破でフェードの簡易演出。
- HP/MP・レベル・経験値・お金・基本コマンド・1〜2種のとくぎ・回復どうぐ、まで。
- セーブは最初はチェックポイント（拠点）方式で十分。

## 世界観（流用前提・コミカル哀愁）
- **物語前提（確定 2026-06-27）＝「定時に帰れない呪い」**：残業の夜、おじさんは謎の力で会社ビルに
  閉じ込められた。出口は消え、エレベーターは見知らぬフロアへ。各階に巣食う“残業の魔物”
  （魔物化した社員・役職者）を倒し、**定時退社＝ビルからの脱出**を目指す。コミカル哀愁トーン。
- 主人公：おじさん（A案）。
- 敵：既存ロスター流用（イモムシ社員・果物メガネ女子・バナナ筋肉女子・大仏豚ボス 等）。
- 王道RPGの役割を“おじさん読み替え”でオリジナル化（拠点＝喫茶店、強敵＝役職者、魔王城＝最上階の社長室）。

## データ駆動の単位（`scaffold/content.example.js` の RPG 例に沿う）
- `ENEMIES`（hp/atk/def/exp/gold/skills/sprite）, `SKILLS`（kind/power/target/element）,
  `ENCOUNTERS`（エリア→敵編成/出現率）, `DIALOGUE`/`SCENARIO`, `ITEMS`。エンジンは薄く、内容はデータで増やす。
